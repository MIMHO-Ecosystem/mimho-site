MIMHO Holder Distribution

The Holder Distribution module is the ecosystem rewards vault: it stores reward allocations and lets eligible holders claim them on-chain, transparently and fairly‚Äîwithout manual intervention.

VIEW CONTRACT
HOLDER DISTRIBUTION MODULE IMAGE

Purpose

Act as a neutral, auditable vault for rewards: allocations are deposited, eligibility is verified, and claims are executed on-chain.

Fairness

Each eligible wallet claims independently. No preferential payouts, no hidden lists, no discretionary withdrawals.

Transparency

All allocations and claims are visible and traceable. Actions are emitted as public events for monitoring and auditing.

Documentation Overview

This module does not promise dates or frequencies. It only guarantees one thing: whenever rewards are allocated, eligible holders can verify and claim them on-chain, with clear, public rules.

How It Works

The contract receives reward deposits and registers allocations. When a holder checks rewards, the contract computes eligibility using on-chain criteria. Claims are executed by the holder wallet calling the claim function‚Äîno one can claim for you.

For security and neutrality, the module is designed so that project wallets and ecosystem contracts cannot withdraw rewards directly. The only movement is: deposit ‚Üí eligible claim.
